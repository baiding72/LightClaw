场景建模与 Runtime 搭建：面向“信息检索—内容整理—待办创建—网页表单填写”个人效率场景，设计轻量级 Agent Runtime，集成 persistent memory、task loop 与 browser / file / search / note / todo / calendar 等工具链；针对复杂多步任务，剥离规划与执行模块，构建闭环的 Plan-Act-Observe-Retry 架构，解决传统 workflow Agent 难以处理的跨页面任务与失败恢复问题。

在线轨迹回流与数据池构建（SFT冷启动）：围绕任务执行中“wrong-tool、GUI click miss、页面状态丢失”等高频失败模式，构建 Gateway + DataPool 机制，自动收集包含 tool call、GUI observation 与 user feedback 的交互轨迹。基于这些数据通过 LoRA 对 Qwen2.5-3B-Instruct 进行行为克隆（SFT），赋予模型基础的工具调用与任务规划能力。

奖励重分配与进度估算（Progress Estimator）：针对长时序任务（如跨页面搜集并填写复杂表单）仅在最终结束才产生反馈的“稀疏延迟奖励”问题，提出基于逐步进展归因（SPA/Step-RL）的奖励重构策略。在基础 LLM 后接入轻量级 MLP 训练进度估算器，将最终任务奖励分解为每步的“进度贡献值”，将稀疏奖励转化为稠密的即时反馈，缓解长链路任务中的 Credit Assignment 难题。

动作校验与 Step-RL 强化学习优化（PPO）：针对 SFT 模型易生成不可执行动作的问题，引入 GUI Grounding 信号（页面加载状态、元素可点击性）作为动作有效性校验，并将其与进度贡献值加权融合成即时奖励。在此基础上，使用 PPO 算法对策略网络进行强化学习迭代，通过 Clip 机制稳定更新，有效减少探索过程中的死循环与无效操作。

系统评测与核心收益：围绕 task success rate、tool execution success rate、recovery rate、GUI grounding accuracy 与 latency 等核心指标建立多维评估体系；经过强化学习对齐迭代，核心场景的多步任务端到端完成率从 85% 提升至 90%，GUI 动作锚定准确率达 96.2%，多轮复杂任务的平均完成时长缩短 28%，用户干预率降至 8% 以下，并具备极强的跨网页泛化能力。
